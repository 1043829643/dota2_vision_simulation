# Dota 2 视野分析前端 PRD

## 1. 产品定位

用户从搜索战队开始，快速理解一支队伍在不同 Patch、对手、阵营和时间段下的视野习惯、常用眼位、有效眼位与风险眼位，并能下钻到单场比赛逐秒验证每一个眼位的实际作用。

本产品不是单局 Match ID 查询器，而是面向教练、分析师、选手和内容复盘者的战队视野研究工具。

## 2. 核心用户

### 2.1 教练 / 分析师

目标：

- 赛前研究某支队伍的常用眼位。
- 找出对手稳定使用的开局眼、中期进攻眼、Roshan 控制眼。
- 判断某些点位是否容易被反制。
- 对比不同 Patch、不同阵营、不同对手下的视野习惯。

### 2.2 选手

目标：

- 快速理解己方或对手的插眼习惯。
- 找到容易被排的眼位。
- 回看某一局具体眼位是否真的产生价值。

### 2.3 内容复盘者

目标：

- 用可视化时间轴解释一场比赛的视野变化。
- 分享单文件报告或截图给观众。

## 3. 核心用户路径

```text
搜索战队
  -> 战队视野主页
  -> 筛选 Patch / 对手 / 阵营 / 时间段
  -> 查看战队眼位地图与排行榜
  -> 点击点位查看详情
  -> 选择具体比赛
  -> 进入单场时间轴逐秒验证
```

辅助路径：

```text
输入 Match ID
  -> 直接进入单场时间轴
```

## 4. 信息架构

### 4.1 Search

首页入口。

用户可以输入：

- 战队 Tag
- 战队名
- 选手名
- Match ID

MVP 先实现战队 Tag 和 Match ID。

搜索结果展示：

- 战队 Tag
- 匹配比赛数
- 最近比赛
- 涉及 Patch
- 可分析眼位数据状态

### 4.2 Team Overview

回答：“这支队伍的视野风格是什么？”

核心指标：

- 比赛数
- 假眼数
- 真眼数
- Hero-seconds
- Unique visible seconds
- Appearances
- Invisible hero-seconds
- Observer contribution
- 高价值眼数量
- 低价值或无收益眼数量

摘要标签：

- 开局眼稳定
- 中期进攻眼多
- 真眼收益高
- 高风险高收益
- 容易被反眼

### 4.3 Ward Map

回答：“这支队伍的眼位主要插在哪里？”

地图模式：

- 活跃眼位时间轴
- 高价值假眼高亮
- 可见敌方英雄点
- 假眼 FoW 视野格
- 真眼范围

交互：

- 播放 / 暂停
- 秒级拖动
- 点击英雄或眼位行后过滤地图
- 只看高价值眼
- 开关假眼视野格

### 4.4 Ward Spots

回答：“这些点位历史表现如何？”

MVP 当前由 `compute_ward_value_metrics.py` 生成静态眼位库报告；后续接入 Web API 后进入主前端。

点位卡片字段：

- 点位 ID
- Ward Type
- Side
- Patch
- 样本数
- 平均价值分
- 被排率
- 平均存活时间
- 真视收益

### 4.5 Leaderboards

回答：“哪些眼位最值得学，哪些应该避免？”

榜单：

- 最常用假眼点位
- 最常用真眼点位
- 最高价值假眼点位
- 最高价值真眼点位
- 最容易被排点位
- 最浪费眼位
- 最稳定点位
- 开局固定眼位
- Roshan 控制贡献最高点位
- 真视收益最高点位

### 4.6 Matches

回答：“这些结论来自哪些比赛？”

字段：

- Match ID
- 日期
- 对手
- 赛事
- Patch
- 阵营
- 比赛时长
- 胜负
- 数据完整性

交互：

- 最近 N 场
- 全选 / 清空
- 点击单局进入时间轴

### 4.7 Match Timeline

回答：“某个眼在具体局内到底发挥了什么作用？”

功能：

- 秒级播放
- 只看当前战队或双方对比
- Observer / Sentry 开关
- 假眼视野遮挡显示
- 真眼无遮挡真视范围
- 动态树木状态影响假眼 FoW
- 当前活跃眼位列表
- 可见敌方英雄列表
- 眼位贡献排行

## 5. 筛选器

全局筛选器影响 Overview、Map、Leaderboards、Matches。

必须支持：

- Patch
- 比赛日期
- 赛事
- 对手
- Radiant / Dire
- 胜负
- 游戏时段
- Ward Type
- 地图区域
- 是否被排
- 最小样本数

MVP 当前实现：

- Team Tag
- Opponent Tag
- Patch
- League
- Match count
- Start second
- End second
- Compare both sides
- Force refresh

## 6. 数据可信度

前端必须持续告诉用户“这份数据有多可信”。

需要展示：

- 样本比赛数
- 眼位数
- 是否有 native FoW
- 是否有动态树木
- 是否有隐身事件
- 是否使用真眼覆盖判断
- 地图版本
- 投影校准版本
- 缓存命中状态

置信度：

- LOW：样本少于 8
- MEDIUM：样本 8 到 19
- HIGH：样本不少于 20

## 9. 当前实现策略

后端已有能力：

- `/api/matches`：按战队查询比赛。
- `/api/visibility`：计算所选比赛的眼位英雄可见数据。
- `/api/ward-value`：计算所选比赛的眼位实例评分、点位聚类、点位详情和排行榜。
- `/resources/*`：提供地图、校准和 FoW 资源。

本次先不重写计算逻辑，只重做 `web/static/index.html`：

- 从“查询表单”改为“战队分析工作台”。
- 保留原有 API。
- 保留地图时间轴能力。
- 修复前端中文显示和信息架构。
- 将 Ward Spots / Leaderboards 接入 Web 页面，用户可以基于当前已选比赛直接计算点位库。

## 10. 已落地的 Web 点位库

当前前端的“点位库与排行榜”页面已经从规划升级为可用功能。

用户操作：

1. 搜索战队。
2. 选择比赛样本。
3. 选择时间段。
4. 点击“计算点位库”。
5. 查看目标战队自己的点位地图、最佳点位、最差点位、最佳假眼实例、最佳真眼实例。
6. 点击地图点位或排行榜行查看点位详情。
7. 从点位实例跳转到单场时间轴对应秒数进行局内验证。

计算规则：

- 只统计当前搜索战队在对应比赛中的己方眼位。
- 复用 native FoW 假眼遮挡。
- 复用动态树木死亡/复活事件。
- 真眼使用 1000 范围无遮挡真视覆盖。
- 隐身英雄必须同时被假眼看到并被己方真眼覆盖，才计入隐身可见收益。
- 点位聚类默认世界坐标半径为 200。

后续增强：

- 按战队阵营过滤点位库。
- 已增加“最常用点位”“稳定高价值点位”“高风险点位”“开局固定点位”“长存活点位”“真视收益点位”等分榜。
- 将点位详情里的实例一键跳转到对应单场时间轴秒数。
- 增加当前筛选状态分享链接。

## 11. 访问方式约束

Web 工作台必须通过本地服务访问：

```powershell
python -m uvicorn web.backend.app:app --reload --host 127.0.0.1 --port 8000
```

然后打开：

```text
http://127.0.0.1:8000/
```

不要直接用 `file://` 打开 `web/static/index.html`，否则浏览器无法访问 `/api/matches`、`/api/visibility` 和 `/api/ward-value`。
