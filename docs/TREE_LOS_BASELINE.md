# Dota 2 树木视野遮挡基准

本文记录当前经过游戏内截图人工校验后，暂定为正确基准的树木 LOS（Line of Sight）遮挡方案。

## 适用范围

- 地图版本：Dota 2 7.41
- 比赛：`8831926213`
- 验证时间：22:38，即 `t=1358`
- 当前网格精度：64 世界单位
- 主要验证眼位：`ehandle=2329196`

## 数据来源

### 眼位

眼位来自数据库中保存的回放解析结果，原始数据来自 `.dem`：

```text
worldX = ward.x * 128 - 16384
worldY = ward.y * 128 - 16384
```

数据库中的真实眼位是权威坐标，不再吸附到附近的 `gridnav`、`no_wards` 或树木合法格。

地形计算使用眼位所在的最近 64 单位网格；树木射线使用未经取整的连续网格坐标。

### 静态树

静态树来自 7.41 地图资源：

```text
maps/dota/entities/default_ents.vents_c
```

反编译后的每棵 `ent_dota_tree` 提供：

- 世界坐标 `origin`
- 模型路径 `model`
- 实例缩放 `scales`
- 旋转 `angles`
- `compile_source_id`
- `hammeruniqueid`

当前地图共提取 2306 棵静态树。

树数据文件：

```text
outputs/resource_probe/tree_collision_candidates.json
```

其中 2298 棵树具有独立树桩模型半径，8 棵特殊树没有树桩模型，回退使用普通模型轴向半径。

## 当前正确基准

### 树木遮挡体

每棵树建模为一个圆形 LOS 遮挡体：

```text
圆心 = 地图资源中的树世界坐标
LOS 半径 = min(stumpRadiusAxis + 30, 128)
```

`stumpRadiusAxis` 已包含地图实例缩放。

`+30` 是根据游戏内截图校准得到的 LOS 外扩余量，不是 Valve 官方公开参数。

### 射线规则

对地形阶段产生的每一个候选可见格：

1. 从数据库真实眼位向目标格中心发射一条连续坐标射线。
2. 检查射线是否与当前可遮挡树的 LOS 圆相交。
3. 当前正式输出将树的 LOS 圆内部视为不可见区域。
4. 如果射线进入树的 LOS 圆，树体格及其后方目标格均不可见。
5. 任意一棵树遮挡该目标格，即删除该目标格。

语义如下：

```text
眼位 -> 可见空间 -> 不可见的树体 -> 不可见的树后阴影
```

树本身、树所在 LOS 区域以及树后阴影均视为不可见。

每个目标格只使用格子中心射线，不采用“多个子采样中任意一条通过就保留整格”的宽松规则。

### 地形与树木的计算顺序

```text
1. 地形高度和高低坡
2. ent_fow_blocker_node
3. 生成候选可见格
4. 独立树木 raycast 过滤
5. 输出最终视野
```

地形和 FOW blocker 由 `rot6` shadowcasting 处理。树木不再加入 `rot6` 的全局角区间合并系统。

## 已修复问题

### 眼位吸附

旧实现会把 `ehandle=2329196` 从原始格 `(212,99)` 移到 `(214,101)`，偏移约 181 世界单位。

现已取消眼位吸附：

```text
原始世界坐标：(3131.1872, -4143.7504)
地形计算格：  (212, 99)
连续射线起点：(212.4248, 98.7539)
```

### 非单调树阴影

旧 `rot6` 树阴影区间合并可能出现：

```text
部分树半径从 128 减到 96，视野反而减少
```

现已改为每个目标格独立执行线段与树圆相交测试。树半径缩小时，可见区域只会增加或保持不变。

### 树体可见性规则

当前正式规则将树木 LOS 圆视为不可见区域：

- 目标格位于树木 LOS 圆内时不可见。
- 射线穿过树木 LOS 圆后，树后目标格不可见。
- 树前且未进入树木 LOS 圆的区域保持可见。

## 已否决方案

以下方案不再作为正确实现：

- 所有树统一使用圆形半径 128。
- 根据树冠视觉包围盒进行 96/128 分档。
- 直接使用树冠视觉包围盒作为 LOS 半径。
- 将每棵树建模成 `128x128` 方形墙。
- 使用 `rot6` 全局角区间合并树木阴影。
- 使用 3x3 子采样，并在任意一条子射线通过时保留整个格子。
- 将回放真实眼位吸附到附近合法网格。
